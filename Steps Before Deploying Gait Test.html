Hexapod Autonomous Nav — Pi Deployment Steps

Date: 2026-03-13  |  Competition: Alamosa CO, April 11 2026 (NASA COSGC)

Prerequisites

  • Raspberry Pi 3B+ with SSH access
  • Arduino Nano connected to Pi via USB
  • Updated offset_full_gait_test_v2.py copied to the Pi

1. Install pyserial

Step 1.1 — SSH into the Pi and install the package:

sudo pip3 install pyserial

Step 1.2 — Verify the installation:

python3 -c "import serial; print(serial.__version__)"

You should see a version number (e.g., 3.5). If you get an ImportError, the install failed.

Why this matters: The nav code does import serial at startup. Without pyserial installed, HAS_SERIAL is set to False and the competition mode falls back to the timed sequence (no sensor-driven navigation).

2. udev Rule for Arduino Nano

Step 2.1 — Find the Arduino's USB vendor/product IDs.

Plug the Arduino Nano into the Pi's USB port, then run:

lsusb

Look for a line containing Arduino, QinHeng, or CH340. Note the ID field.

ChipTypical IDDescription
CH3401a86:7523Clone Nano (most common)
ATmega16U22341:0043Genuine Arduino

Step 2.2 — Create the udev rule file:

sudo nano /etc/udev/rules.d/99-arduino.rules

Paste this single line (replace vendor/product IDs if yours differ):

SUBSYSTEM=="tty", ATTRS{idVendor}=="1a86", ATTRS{idProduct}=="7523", SYMLINK+="ttyACM_ARDUINO", MODE="0666"

Save and exit (Ctrl+O, Enter, Ctrl+X).

ParameterWhat it does
SYMLINK+="ttyACM_ARDUINO"Creates a stable device name at /dev/ttyACM_ARDUINO regardless of USB plug order
MODE="0666"Allows any user to read/write without sudo

Step 2.3 — Reload udev rules:

sudo udevadm control --reload-rules
sudo udevadm trigger

Step 2.4 — Verify. Unplug and replug the Arduino, then:

ls -la /dev/ttyACM*

You should see both /dev/ttyACM0 and the symlink /dev/ttyACM_ARDUINO.

Step 2.5 (Optional) — Use the stable symlink in the gait file.

The nav code currently uses ARDUINO_PORT = "/dev/ttyACM0". If you want to use the stable symlink, change it to:

ARDUINO_PORT = "/dev/ttyACM_ARDUINO"

This is optional. If only the Arduino is on USB (besides the servo debug board on /dev/ttyUSB0), ttyACM0 will be consistent.

3. Quick Verification Checklist

After completing steps 1 and 2, run these checks on the Pi:

# Check pyserial
python3 -c "import serial; print('pyserial OK:', serial.__version__)"

# Check Arduino port
ls -la /dev/ttyACM*

# Test Arduino communication (prints 20 frames, no servos needed)
sudo python3 offset_full_gait_test_v2.py --test-arduino

# Dry-run nav (sensors active, speed=0, robot on stand)
sudo python3 offset_full_gait_test_v2.py --competition-dry-run

4. Competition Day Commands

CommandPurpose
sudo python3 offset_full_gait_test_v2.py --competitionFull autonomous nav with sensors
sudo python3 offset_full_gait_test_v2.py --competition-dry-runSensors active, speed=0, logs only
sudo python3 offset_full_gait_test_v2.py --test-arduinoPrint 20 Arduino frames (no Heart)
sudo python3 offset_full_gait_test_v2.py --test-competitionTimed fallback (no sensors)
Graceful degradation: If pyserial is missing or the Arduino isn't connected, --competition automatically falls back to the timed sequence (quad@400 for 45s, then wave@350 for 30s). No crash, no error — just no sensor-driven steering.
