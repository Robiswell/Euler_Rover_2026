System Overview

The RHex Hexapod Rover is a six-legged autonomous robot designed for the NASA COSGC competition in Alamosa, Colorado. It navigates a 15–20 ft course over wet sand, loose rocks, and ramps using sensor-driven obstacle avoidance. The system comprises two programs running on separate processors:

Pi Gait Engine — final_full_gait_test.py

A single-file gait engine for a six-legged rover built on STS3215 servos running in velocity mode. No position setpoints are issued during walking. A Buehler clock drives each leg through stance and air phases; LERP smoothing prevents jerky gait transitions.

Two-Process Architecture (Brain / Heart)

The gait engine uses Python's multiprocessing module to run two cooperating processes. They share state exclusively through mp.Value and mp.Array primitives — no queues, no pipes, no threads (except the Arduino reader daemon thread in Brain).

Brain Main Process

  • Mission sequencer & state machine
  • Autonomous navigation FSM (8 states, 14 priorities)
  • Sets shared variables for Heart to read
  • Reads Arduino sensor data via daemon thread
  • Handles KeyboardInterrupt and emergency stop
  • Runs recovery behaviors (wiggle, self-right roll)
  • Calls tsleep() / stall_tsleep() for timed phases

Heart Subprocess (gait_worker())

  • 50 Hz kinematics loop (20ms target period)
  • Reads shared variables each tick
  • Buehler clock phase computation per leg
  • LERP smoothing of all parameters
  • Feedforward + phase feedback velocity calculation
  • GroupSyncRead: positions + loads every tick
  • GroupSyncWrite: goal speeds every tick
  • Safety governor (temp, voltage, stall, bus disconnect)
  • Rotating telemetry reads (temp/voltage/current)
  • Homing sequence on shutdown

Process Lifecycle

  1. Init: init_and_align_servos() snaps all legs to home position via position mode, then switches to velocity mode.
  2. Heart spawned: mp.Process(target=gait_worker, ...)
  3. Heartbeat guard: Brain waits for shared_heartbeat != 0 with a 15-second timeout. Uses != 0 instead of < 50 to avoid overflow false-pass on ~497-day uptime.
  4. Mission: Brain runs the selected mission mode (demo, competition, test).
  5. Shutdown: is_running = False → Heart exits loop → homing sequence → join(timeout=5) → terminate() if still alive.

Heart Loop Timing

Target: 50 Hz (20 ms per tick). Uses a hybrid sleep strategy:

  • time.sleep(sleep_time - 0.002) for coarse sleep
  • Busy-wait spin loop for the final 2 ms to hit the target precisely
  • Overrun detection: if loop_dt_ms > 18.0, logs [OVERRUN] tag
  • After 3 consecutive overruns, flushes ring buffer to disk
  • os.nice(-20) for real-time priority (requires sudo)
  • GC disabled after initial collection to prevent jitter

Gait Definitions

Three gait patterns are defined, each with a duty cycle (fraction of time each leg spends in stance) and per-servo phase offsets (0–1, fraction of cycle).

Tripod (Gait 0) — Duty 0.50

Two groups of 3 legs, 180° apart. At any instant, 3 legs are in stance and 3 in air. The fastest gait but least stable.

Servo264135
Offset0.00.00.00.50.50.5

Groups: {2, 6, 4} and {1, 3, 5} alternate.

Warning: Do NOT use tripod on slopes > 10°. Peak servo torque exceeds rated 10 kg·cm. Use Wave with COG shift for ramps.

Wave (Gait 1) — Duty 0.75

Each leg spends 75% in stance. Six legs offset by 1/6 cycle (~0.167) so exactly one leg is in the air at any moment. Slowest but most stable — maximum ground contact at all times.

Three diagonal pairs, each 1/3 cycle apart. Good balance of speed and stability. Default gait for autonomous navigation.

Buehler Clock Math

The Buehler clock maps a normalized time value [0, 1) to a leg angle [0°, 360°). It handles wrap-around geometry for stutter-free rotation.

get_buehler_angle(t_norm, duty_cycle, start_ang, end_ang, is_reversed)

Inputs:

  • t_norm — normalized phase time [0, 1) for this leg
  • duty_cycle — fraction of cycle in stance (0.5, 0.7, or 0.75)
  • start_ang / end_ang — stance impact window in degrees
  • is_reversed — True when leg_hz < 0 (backward motion)

Math:

  1. Compute stance_sweep as the shortest signed arc from start_ang to end_ang: 
    stance_sweep = (end_ang - start_ang + 180) % 360 - 180
  2. If reversed: swap start/end, negate stance_sweep.
  3. Stance phase (t_norm ≤ duty): linear interpolation through the stance arc. 
    angle = start_ang + stance_sweep * (t_norm / duty_cycle)
  4. Air phase (t_norm > duty): linear interpolation through the remaining arc. 
    angle = end_ang + air_sweep * ((t_norm - duty) / (1 - duty))

The air_sweep is always 360 - |stance_sweep| with matching sign, ensuring the leg sweeps through the full 360° in one cycle.

LERP Smoothing

Every Heart tick, all parameters are smoothed via linear interpolation to prevent jerky transitions when Brain writes new values. The LERP rate is tied to real elapsed time for loop-rate invariance.

lerp_rate = min(1.0, 4.0 * real_dt)   # ~0.08 per tick at 50Hz, reaches 1.0 in ~250ms

smooth_hz   += (target_hz   - smooth_hz)   * lerp_rate
smooth_turn += (target_turn - smooth_turn) * lerp_rate

Circular LERP for Angles

Impact window angles require circular (wrapping) interpolation to prevent spinning the wrong way around when crossing the 0°/360° boundary:

d_s = (target_start - smooth_start + 180) % 360 - 180
smooth_start = (smooth_start + d_s * lerp_rate) % 360

The (+180) % 360 - 180 pattern extracts the shortest signed angular difference.

Phase Offset LERP

Each servo's phase offset is individually LERPed with the same circular wrap pattern, enabling smooth gait transitions (e.g., tripod → wave) without any leg snapping to a new position.

Duty Cycle LERP

Duty cycle is LERPed linearly at the same 4× rate, smoothing transitions between gaits with different duty cycles.

Seeding: smooth_offsets is initialized from GAITS[0]['offsets'] (tripod) at startup so all legs are correctly phased from tick 1, avoiding the all-6-legs-in-phase cold-start problem.

Feedforward Velocity Calculation

The feedforward computes the servo speed needed to traverse the current phase arc in exactly the right amount of time:

if t_leg <= smooth_duty:
    deg_per_sec = (abs(stance_sweep) * abs(leg_hz)) / smooth_duty
else:
    deg_per_sec = (abs(air_sweep) * abs(leg_hz)) / (1.0 - smooth_duty)

ff_speed = min(499.0, deg_per_sec * VELOCITY_SCALAR)

The formula is: arc_degrees × cycle_hz / duty_fraction. Multiplied by VELOCITY_SCALAR = 1.85 to convert deg/s to raw STS3215 speed units.

Speed cap: Capped at 499.0 STS (the STS3215 no-load ceiling is ~270 deg/s × 1.85 = 499.5 STS). Commanding higher speeds causes phase lag, wrong turn directions, and chassis ground strikes.

Phase Feedback (KP_PHASE)

A proportional controller corrects phase tracking errors:

error       = target_phase - current_phase
short_error = (error + 180) % 360 - 180    # shortest signed angular error
raw_speed   = ff_speed + (short_error * KP_PHASE)

With KP_PHASE = 12.0, a 10° phase error produces a 120 STS correction.

Air-Phase Sign Guard

During the air phase, the leg moves fast in the return direction. Without correction, the shortest-path error calculation can produce a backward signal. A 10° linear transition blend prevents this phase reversal artifact:

if t_leg > smooth_duty:
    if not is_rev_leg and short_error < -80:
        depth = -80.0 - short_error
        blend = min(1.0, depth / 10.0)
        short_error += 360 * blend

Speed Governor

Prevents air-phase speed from exceeding STS3215 mechanical limits:

max_safe_hz = (2800.0 / VELOCITY_SCALAR * (1 - smooth_duty)) / max(5.0, abs(air_sweep))
hz_L = clamp(hz_L, -max_safe_hz, max_safe_hz)
hz_R = clamp(hz_R, -max_safe_hz, max_safe_hz)

The 2800 constant represents the maximum safe STS3215 speed in raw units. The governor is computed dynamically based on current duty cycle and air sweep angle.

Left/Right Clock Resync

When going straight (no turn bias), the left and right master clocks can drift apart. A gradual decay brings them back together:

if abs(smooth_turn) < 0.01 and abs(smooth_hz) > 0.01:
    clock_diff = (master_time_L - master_time_R + 0.5) % 1.0 - 0.5
    if abs(clock_diff) > 0.005:
        decay = clock_diff * min(1.0, 2.0 * real_dt)
        master_time_L -= decay / 2
        master_time_R += decay / 2

Gradual decay (not snap) prevents sudden leg jerks. Only fires when going straight and both sides spinning the same direction.

Safety Systems

Safety Temperature Protection

  • Per-servo tracking: Individual temp_spike_counters per servo ID, not a single global counter.
  • Rotating reads: One servo per tick (6-tick cycle) to minimize bus traffic.
  • Ghost temp filter: If temp reads exactly 125°C (GHOST_TEMP), it's an EMI bus artifact — latches a flag, skips counter update.
  • Hysteresis: Counter increments if temp > TEMP_MAX (65°C); decrements by 1 (not instant reset) otherwise.
  • Shutdown: When worst-servo counter exceeds 15, triggers safety shutdown with hottest servo ID logged.

Safety Voltage Protection

  • volt_dip_counter increments when voltage < VOLTAGE_MIN (10.5V), decrements by 1 otherwise (hysteresis).
  • Shutdown when counter exceeds 150 (150 ticks = 3s debounce at 50 Hz).

Safety Stall Detection

  • Latching 3-frame logic: Counter increments when load magnitude > STALL_THRESHOLD (750), decrements by 1 otherwise.
  • Stall latches when counter ≥ 3, clears when counter reaches 0.
  • During stall_override = True: counter zeroed, is_stalled forced False.
  • State transitions (enter/exit stall) logged with ms-precision timestamp, load, threshold, position, speed.

Safety Overload Prevention (TE Cycling)

  • Time-based (1.5s): When load exceeds STALL_THRESHOLD for 1.5 continuous seconds.
  • Action: Writes TE=0, MODE=1, TORQUE_LIMIT=1000, waits 100ms (5-tick dwell), then TE=1.
  • EEPROM protection: Max 10 cycles per servo per session (OVERLOAD_MAX_CYCLES).
  • Decoupled from stall_override: Fires even during self-right roll to prevent hardware overcurrent lockout.
  • Margin: 21% below the STS3215 2.0s hardware cutoff, regardless of loop rate.

Safety Bus Disconnect Detection

  • comm_fail_streak counts consecutive ticks where zero servos respond to position GroupSyncRead.
  • After 10 consecutive zero-response ticks (200ms): logs [BUS-DISCONNECT], flushes ring buffer, sets is_running = False.
  • Prevents servos spinning indefinitely on USB cable disconnect.

Safety Per-Servo Communication Failure

  • servo_comm_fails[sid] increments when gsread_pos.isAvailable() fails for that servo.
  • At 50 consecutive failures (1s): prints warning, sets servo_disabled[sid] = True → Heart zeroes that servo's speed.

Safety Safety Shutdown Action

On any safety trigger (overtemp, undervolt, bus disconnect):

  1. Log the specific reason with trigger values
  2. Flush full 100-entry ring buffer to disk for post-mortem analysis
  3. Set is_running = False
  4. Heart exits loop and runs homing sequence

Steering Modes

Carve Turn (Differential Speed)

Brain sets shared_turn_bias to a non-zero value while speed > 0. Heart computes:

hz_L = smooth_hz + smooth_turn
hz_R = smooth_hz - smooth_turn

Both sides move forward, one faster — the robot follows a gradual arc.

Pivot Turn (Rotate in Place)

Brain sets speed = 0 and turn_bias = ±value. One side goes forward, the other backward — the robot rotates around its center.

Reverse

Setting shared_x_flip = -1 flips the sign of target_hz, reversing all legs. A decel pause (speed=0 for 0.5s) is required before reversing to prevent LERP bleed from the previous direction.

Navigation Arc Turn

During autonomous navigation (P11), turn intensity is computed from the diagonal sensor difference:

turn_intensity = tanh((FDL - FDR) / 50.0)

Positive = left side is freer = turn left. Applied as turn = ±abs(turn_intensity) × MAX_TURN_BIAS.

Recovery Behaviors

Wiggle — state_recovery_wiggle(duration=10)

Breaks static friction and unsticks legs from sand by oscillating forward/backward rapidly.

  1. Logs stall bitmap and servo loads
  2. Sets stall_override = True (prevents stall detection from zeroing wiggle motion)
  3. Saves current gait; switches to Tripod (gait 0, 50% duty, best wiggle effectiveness)
  4. Resets impact window to 330/30 (avoids inheriting stale COG-shift window)
  5. Sets speed = 350
  6. Oscillates: x_flip = -1 for 0.3s, x_flip = 1 for 0.3s, repeating until duration expires
  7. Finally: speed=0, x_flip=1, stall_override=False, restores saved gait

stall_tsleep(duration, wiggle_threshold=3)

Wraps long Brain sleeps in 3-second chunks. After each chunk:

  1. Counts servos with load > STALL_THRESHOLD
  2. If stalled ≥ threshold and not in a pivot turn:
    • Saves speed, turn_bias, impact_start/end, x_flip
    • Halts (speed=0) immediately before calling wiggle
    • Calls state_recovery_wiggle()
    • Restores all saved values
    • One immediate retry if still stalled after first wiggle
  3. Pivot guard: Does not trigger during pivot turns (high load is by design)

Self-Right Roll — state_self_right_roll()

Momentum roll to flip a capsized robot upright. Unvalidated on hardware.

Safety Guards (checked in order):

  1. Orientation guard: Switches to wave gait, waits 1.5s for LERP, counts servos with load > 200. If ≥ 4 loaded → robot is upright → skip roll.
  2. Voltage pre-check: If shared_voltage < 11.0V, skip — self-right draws ~4.9A and voltage sag could trigger safety shutdown mid-roll.
  3. Sets stall_override = True (allows legs to push hard without stall detection)
  4. Sets roll_mode = True (bypasses left-servo negation so all legs spin same physical direction)

Sector Classification

Three sectors are classified by taking the worst-case (maximum danger level) across their constituent sensors:

  • Front: max(FDL, FCF, FDR)
  • Left: max(FDL, RDL)
  • Right: max(FDR, RDR)

Cliff Detection (CliffDetector)

Dual front/rear cliff detection using FCD and RCD (down-facing sensors):

  • EMA tracking: Exponential moving average of ground distance (alpha = 0.2, initial = 15 cm)
  • EMA update: Applied before candidate check for readings in (0, 40]
  • Cliff candidate: reading > 30 cm (absolute threshold) OR reading > EMA + 10 cm (adaptive delta)
  • Confirmation: 2 consecutive candidate frames required to declare a cliff
  • Blind zone (reading = -1): Very close object, not a cliff — resets counter

Upside-Down Sensor Remapping

When UpsideDown = 1, the parser swaps front and rear sensor arrays (dists[0:4] ↔ dists[4:8]) so the FSM's "front" and "rear" labels match physical reality regardless of orientation.

Telemetry & Logging (4-Tier System)

Tier 1: In-Memory Ring Buffer (50 Hz)

A collections.deque(maxlen=100) (~8 KB) storing 23 fields per entry. Written every tick, flushed to disk only on events (bus disconnect, safety shutdown, overruns). Provides 2-second post-mortem context.

Tier 2: 1 Hz Heartbeat Log

Comprehensive telemetry written once per second:

timestamp | G:gait Spd:N Trn:+/-N | V:N.NN Vc:NNN | T:temps Tc:NN
| L:loads | Stall:sN | A:amps | Loop:ms
| Pdelta Jit:max/std Gov:% TE:cycles PhErr:+/-deg Comm:N StDur:dur

Includes special tags: [GHOST] (125°C artifact), [V_WARN:N] (dip counter > 100), [T_WARN:N] (spike counter > 10).

Tier 3: Event Logging

  • Stall events: Buffered in _stall_log_buffer, flushed at shutdown. Fields: timestamp, ENTRY/EXIT, servo, load, threshold, concurrent count, position, cmd_speed, ff_speed, duration, te_fired.
  • TE cycle events: Immediate write. Fields: timestamp, [TE-CYCLE], servo, stall_dur, load, cycle_n/max.
  • Safety shutdown: [SHUTDOWN] tag with trigger reason + per-servo temps and loads + full ring buffer dump.
  • Bus disconnect: [BUS-DISCONNECT] tag with streak count and last known positions.
  • Loop overruns: [OVERRUN] tag with loop_dt_ms; 3 consecutive triggers ring buffer flush.
  • Brain state transitions: set_gait_state() logs before → after diffs for every changed variable.
  • Recovery events: [WIGGLE-START], [WIGGLE-END] with stall IDs, loads, duration, outcomes.

Tier 4: Session Headers

  • Heart header: hostname, Python version, PID, all constants, voltage, home positions, bus config.
  • Brain header: timestamp, mission phases description.

Log rotation: When the log file exceeds 10 MB (LOG_MAX_SIZE), it is renamed to .timestamp.old.

GroupSyncRead / GroupSyncWrite Patterns

GroupSyncWrite — Goal Speeds (every tick, all 6 servos)

  • Encoding: positive speed → abs_speed as-is; negative speed → abs_speed | 0x8000 (STS sign bit)
  • clearParam() called every tick after txPacket()
  • Left servos are negated at command stage (raw_speed = -raw_speed) to produce matching physical rotation despite mirror mounting

GroupSyncRead — Present Position (every tick, all 6 servos)

  • Angle computed from encoder diff from HOME, applying DIRECTION_MAP
  • On read failure: uses last_actual_phases[sid] (holds last known value)

GroupSyncRead — Present Load (every tick, all 6 servos)

  • Load magnitude extracted as load & 0x3FF (10-bit; direction bit in bit 10)
  • Used for stall detection and shared_servo_loads

Individual Reads — Temp / Voltage / Current

  • Rotating single-servo reads: one servo per tick (6-tick cycle)
  • Reduces bus traffic vs. GroupSyncRead for data needed from only one servo at a time

Servo Initialization & Homing

Startup Sequence (init_and_align_servos())

  1. Open serial port with retry (30 attempts, 1s between, prints every 5th)
  2. For each servo: TE=0, MODE=0 (position), ACCEL=20, TORQUE_LIMIT=400, GOAL_SPEED=800, GOAL_POSITION=HOME, TE=1
  3. Wait 3.0s for all legs to reach home
  4. For each servo: TE=0, MODE=1 (velocity), ACCEL=0, TORQUE_LIMIT=1000, TE=1
  5. Pre-register all servos for GroupSyncRead

Shutdown Homing (Heart finally block)

  1. GOAL_SPEED=0 for all servos (halt in place)
  2. Wait 0.1s
  3. For each servo (same order as init): TE=0, MODE=0, ACCEL=20, TORQUE_LIMIT=400, GOAL_SPEED=800, GOAL_POSITION=HOME, TE=1
  4. Wait 3.0s for legs to reach home
  5. For each servo: TE=0 (release torque)
  6. Close serial port

Ultrasonic Measurement — measureClassified()

Single-pass, single-sensor measurement. No partial echoes.

  1. Drain: Wait for ECHO pin to go LOW (up to 40ms). If stuck HIGH → sensor saturated → return -1.0 (VERY NEAR).
  2. Trigger: Send 10µs pulse on TRIG pin.
  3. Rising edge: Wait for ECHO to go HIGH. Timeout (20ms) → no response → return 300.0 (VERY FAR).
  4. Falling edge: Measure HIGH duration. Timeout → return 300.0 (VERY FAR).
  5. Blind zone check: Duration < NEAR_RT_US (~175 µs, equivalent to 3 cm round-trip) → return -1.0.
  6. Distance conversion: d = duration × 0.0343 × 0.5 (speed of sound at ~20°C).
  7. Range clamp: < 2 cm → 2.0; > 300 cm → 300.0.

IMU Configuration (BNO085)

  • Interface: I2C at address 0x4A
  • Reports enabled (50 Hz each):
    • Game Rotation Vector — boot-relative quaternion, no magnetometer (avoids 180° flip from upside-down mounting)
    • Accelerometer
    • Gyroscope
    • Gravity vector — used for UpsideDown detection
  • Quaternion access: getQuatI(), getQuatJ(), getQuatK(), getQuatReal() — shared rawQuat fields serve whichever rotation vector type is enabled
  • I2C timeout: 3ms with bus reset on timeout (Wire.setWireTimeout(3000, true))
  • IMU reinit: After 50 consecutive data misses, attempts full re-initialization

UpsideDown Detection

Uses the gravity report (not raw accelerometer) for robust upside-down detection:

upside_down_flag = (last_gravity_z > 5.0f) ? 0 : 1;
  • gravity_z > 5.0: Rover is right-side up (gravity points "up" in IMU frame) → UpsideDown = 0
  • gravity_z ≤ 5.0: Rover is physically inverted → UpsideDown = 1

IMU Data Handling

  • Last-known values: All IMU values stored in last_* variables. If no new data is available on a loop iteration, the previous values are reused in the CSV output. This ensures every loop produces a CSV row.
  • Drain loop: while (imu.dataAvailable()) drains all pending reports so getters return the most recent values.
  • Fail counter: imu_fail_count tracks consecutive misses. At 50 misses (IMU_FAIL_RESET), attempts full IMU re-initialization.

Special Pin Handling

Pin 13 (RCF ECHO): Uses INPUT_PULLUP instead of INPUT to counteract the onboard LED load on the Arduino Nano, which can cause false LOW readings on this pin.

PROGMEM Optimization

Sensor name strings are stored in program memory (PROGMEM) instead of SRAM, saving approximately 130 bytes of RAM. Names are read via strcpy_P() from a PROGMEM pointer table.

CSV Output Format

timestamp_ms, FDL, FCF, FCD, FDR, RDL, RCF, RCD, RDR,
QuatW, QuatX, QuatY, QuatZ, AccelX, AccelY, AccelZ,
GyroX, GyroY, GyroZ, UpsideDown
  • 20 columns matching the Python parser's expected format
  • Distances: 2 decimal places
  • Quaternions: 6 decimal places
  • Accelerometer/Gyro: 4 decimal places
  • Timestamp: Captured after measurements complete (reflects when data is ready, not when the loop started)
  • Header row: Printed once at startup with sensor names and column labels

Loop Timing

With INTERVAL = 0, the loop runs as fast as sensor measurements allow. Each of the 8 sensors takes up to 20ms (TIMEOUT_TOTAL) plus a 2ms inter-sensor gap, giving a worst-case measurement window of ~176ms. Typical rate is approximately 5 Hz.
